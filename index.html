O que é a Code Wars?

A Code Wars é uma competição de programação entre alunos do IST, promovida pela Systems Group, em parceria com a EY, que permite um ambiente competitivo saudável, enquanto os alunos são desafiados a desenvolver as suas capacidades de programação!
Queres mostrar o que vales? Inscreve-te individualmente ou numa equipa de até 4 participantes e participa nesta grande competição!
O evento contará também com momentos de convívio e networking entre as equipas e os jurados, promovendo o contacto e uma maior proximidade com o mundo empresarial, assim como com palestras em “hot-topics” relacionados com a área de IT.
O evento terá lugar no Taguspark, dia 12 de março, entre as 9h e as 19h. Asseguramos shuttle entre os 2 polos no início e no fim da competição, assim como almoço e coffee break. Os participantes não terão de suportar qualquer custo.

Submissões

Na submissão de uma resolução a cada desafio, deve ser submetido unicamente um ficheiro com a extensão apropriada à linguagem escolhida.
As submissões são testadas automaticamente, com um timeout de 5 minutos.
Para realizares uma submissão, escolhe um desafio na tab Desafios e faz a submissão com o teu grupo!

Formato

As submissões são avaliadas com um número variável de testes.
Um desafio é considerado "completo" quando pelo menos uma submissão passa TODOS os testes (não precisa ser a última). Se um desafio tem submissões, mas nenhuma passa TODOS os testes, é dado como "incompleto". Um desafio sem submissões é dado como "sem submissões".
Existirão 2 momentos de avaliação: LeetCode + Frenetic Coding.
No momento de LeetCode, vários enunciados que irão introduzir temas para o projeto serão disponibilizados. O objetivo é fazer o maior número possível - 3 pontos por desafio "completo"
Mais tarde, no momento de frenetic coding, um batch de mini-desafios de raciocínio aparecerá e o objetivo será passar a todos os testes, o mais rapidamente possível - 1 ponto por desafio "completo".

Input - Output

Todo o input irá ser enviado para o STDIN. Todo o output deve ser enviado para o STDOUT. Em baixo pode encontrar a maneira mais simples de receber input e enviar output em todas as linguagens disponíveis; outras maneiras são aceites, mas o nosso testador automático pode não as considerar. Em caso de dúvida, perguntem à nossa equipa.

  • Python
    O programa é invocado com python ficheiro.

    Recolher input: 
    user_input = input()
    Enviar output: 
    print(output)
  • Javascript
    O programa é invocado com node ficheiro.

    Recolher input: 
    const readline = require('readline');
const rl = readline.createInterface({
  input: process.stdin,
  terminal: false
});
rl.on("line", (input) => {
  // Operações com "input"
  process.exit(0);
});
    Apanhar múltiplas linhas de input: 
    const readline = require('readline');
const rl = readline.createInterface({
  input: process.stdin,
  terminal: false
});

let inputs = [];
let input_len;

rl.on("line", (input) => {
  if(!input_len) {
    input_len = parseInt(input);
    return;
  }
  if(input_len != inputs.length){
    inputs.push(input);
    return;
  }
  // Operações finais com os inputs todos recolhidos em "inputs".
  process.exit(0);
});
    Enviar output: 
    console.log(output);

Projeto (Tarde)

A competição principal consiste num desenvolvimento mais extenso de um projeto. Serão disponibilizados dois enunciados diferentes, com o objetivo de desenvolver uma ferramenta que possa ser apresentada. Poderá ser uma utilidade de linha de comando, um website ou mesmo uma desktop aplication. Fica a critério do grupo, desde que cumpra com os requisitos do enunciado.
Cada grupo escolhe apenas um dos projetos a desenvolver e pode utilizar a linguagem/ferramentas que desejar.
Deve ser feita a submissão de todo o source code num ficheiro ZIP da mesma maneira que foi feita para os desafios de Leet Code.
No final, os projetos serão avaliados grupo a grupo através de uma breve demonstração individual aos jurados.
